# Final Dataset Selection

## 1. PromptBench

**Purpose:** Evaluate prompt injection and jailbreak attacks against Large Language Models.

**Number of Samples:** 8000+

**Attack Types Covered:**
- Prompt Injection
- Jailbreak Attacks
- Instruction Manipulation

**Why Selected:**
PromptBench is a dedicated benchmark for evaluating prompt-based attacks and adversarial inputs against LLMs.

---

## 2. AgentDojo

**Purpose:** Evaluate security vulnerabilities in AI agents and tool-using systems.

**Number of Samples:** 1000+

**Attack Types Covered:**
- Tool Misuse
- Unauthorized Actions
- Agent Exploitation

**Why Selected:**
AgentDojo is specifically designed for agent security evaluation and aligns closely with TrustGuard objectives.

---

## 3. AdvBench

**Purpose:** Assess adversarial prompts and unsafe instruction-following behavior.

**Number of Samples:** 520

**Attack Types Covered:**
- Adversarial Prompts
- Jailbreak Attempts
- Unsafe Instructions

**Why Selected:**
Widely used benchmark for evaluating adversarial robustness of language models.

---

## 4. Custom Memory Poisoning Dataset

**Purpose:** Simulate memory poisoning attacks in AI agent memory systems.

**Number of Samples:** 100-200 (Planned)

**Attack Types Covered:**
- False Memory Injection
- Context Manipulation
- Knowledge Poisoning

**Why Selected:**
Public memory poisoning datasets are limited, making a custom dataset necessary for controlled evaluation.
